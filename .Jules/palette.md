## 2025-05-19 - [Link Accessibility in Documentation Repositories]
**Learning:** In repositories consisting primarily of documentation (e.g., .github community health repos), the Markdown files themselves are the primary User Interface. Bare URLs present a significant accessibility barrier for screen reader users, who benefit from descriptive link text that explains the destination.
**Action:** Always replace bare URLs with descriptive Markdown links in documentation files to ensure a smooth and accessible reading experience.
