## 2025-05-14 - [Markdown Accessibility: Bare URLs]
**Learning:** In documentation-only repositories, the Markdown files themselves are the primary User Interface. Bare URLs are an accessibility issue as they lack context and are tedious for screen reader users to hear read aloud character-by-character.
**Action:** Always replace bare URLs (e.g., `<https://example.com>`) with descriptive link text (e.g., `[Example Website](https://example.com)`) to provide better context and improve the experience for users with assistive technologies.
